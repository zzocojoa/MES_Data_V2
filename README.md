# 스마트팩토리 MES Data 파이프라인 V2

이 프로젝트는 현장의 제조 데이터(PLC, SPOT 온도, 작업 일지)를 Synology NAS DB
공간으로 수집하고, 이를 실시간 모니터링하기 위한 대시보드를 제공하는 4계층(DB,
수집기, 백엔드, 프론트엔드) 아키텍처입니다.

---

## 🚀 개발자 실행 가이드

본 프로젝트는 백엔드 서버(데이터 제공)와 프론트엔드 서버(화면 렌더링)가 분리되어
동작합니다. 대시보드를 띄우기 위해서는 두 서버를 모두 실행해야 합니다.

각 폴더에 마련된 원클릭 배치(`.bat`) 파일을 사용하거나, 터미널에서 직접 명령어를
입력해 실행할 수 있습니다.

### 1. 백엔드 (API 서버) 실행 방법

백엔드 서버는 `api_server` 폴더에 위치하며, FastAPI를 사용해 8000번 포트에서
동작합니다. NAS DB에서 데이터를 읽어 프론트엔드에 전달합니다.

- **가장 쉬운 방법 (원클릭):** `api_server/run_server.bat` 파일 더블 클릭 실행
- **터미널(개발자) 직접 실행:**
  ```bash
  # 1. 터미널 열기 및 해당 폴더로 이동
  cd api_server

  # 2. 파이썬 가상환경 활성화
  venv\Scripts\activate

  # 3. FastAPI 서버 실행 (자동 재시작 모드)
  uvicorn main:app --host 0.0.0.0 --port 8000 --reload
  ```
  ✅ **접속 확인:** `http://localhost:8000/docs` 에서 실시간 API 명세서 확인
  가능

### 2. 프론트엔드 (웹 대시보드) 실행 방법

프론트엔드는 `frontend` 폴더에 위치하며, React(Vite 기반)를 사용해 5173번
포트에서 동작합니다. **반드시 백엔드가 먼저 켜져 있어야 에러가 나지 않습니다.**

- **가장 쉬운 방법 (원클릭):** `frontend/run_frontend.bat` 파일 더블 클릭 실행
- **터미널(개발자) 직접 실행:**
  ```bash
  # 1. 새 터미널 창을 열고 해당 폴더로 이동 (백엔드 터미널과 별도)
  cd frontend

  # 2. Vite 개발 서버 실행
  npm run dev
  ```
  ✅ **접속 확인:** `http://localhost:5173` 을 브라우저에서 열어 모니터링
  대시보드 확인

### (참고) 파이썬 실시간 데이터 수집기 (Data Collector)

공장에서 들어오는 실시간 데이터 CSV 파일을 스스로 감지하고 DB에 1초 만에 밀어
넣는 데몬 스크립트입니다. `ext_pipeline` 폴더에 위치합니다.

- **실행 방법:** `ext_pipeline/run_watcher.bat` 더블 클릭
- **사용법:** 위 스크립트를 켜둔 상태로, `ext_pipeline/data_in` 폴더에 CSV
  파일(PLC, 온도, 일지 등)을 넣으면 즉시 파싱되어 NAS DB로 전송됩니다.
