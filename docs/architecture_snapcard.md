# 스마트팩토리 데이터 파이프라인 아키텍처 (스냅카드 플랜)

기존의 무겁고 얽혀있던 Supabase 플랫폼을 걷어내고, 역할을 명확하게 나눈 **MSA(Microservice Architecture) 방식의 4단계 스냅카드 플랜**입니다. 각 파트는 철저하게 자신의 책임을 다하며, 서로 독립적으로 개발 및 유지보수가 가능합니다.

---

## 🗄️ [CARD 1] 데이터베이스 (DB / NAS)
**"오직 데이터를 빠르고 안전하게 담아두는 거대한 순수 창고"**

*   **위치:** 시놀로지 NAS (192.168.0.235)
*   **기술 스택:** 순정 PostgreSQL 15 (Docker 컨테이너 1개)
*   **핵심 역할 (R&R):**
    *   파이썬 업로더가 쏘아주는 수만 건의 PLC/온도/일지 데이터를 초고속으로 적재
    *   프론트엔드/백엔드에서 데이터 조회 요청이 오면 SQL 쿼리에 맞춰 데이터를 반환
    *   (제거된 역할: 사용자 로그인 인증, 자동 API 생성, 웹사이트 서빙 등은 일절 하지 않음)

---

## 🚚 [CARD 2] 데이터 콜렉터 (Data Collector / Edge)
**"현장 장비의 CSV를 읽어 NAS 창고로 1초 만에 배달하는 화물 트럭"**

*   **위치:** 공장 현장 PC (로컬 윈도우)
*   **기술 스택:** Python (`psycopg2` 또는 `SQLAlchemy`, `pandas`) / Windows 스케줄러
*   **핵심 역할 (R&R):**
    *   **CSV 파싱:** 주기적으로(또는 실시간) 생성되는 압출기(PLC), 열화상(SPOT) CSV 파일 읽기
    *   **데이터 정제:** 분석된 스키마에 맞춰 시간 변환(UTC/KST), 결측치 처리, 컬럼 매핑
    *   **초고속 적재 (Bulk Insert):** 과거처럼 500개씩 API로 빙빙 돌지 않고, 다이렉트 SQL 연결(`COPY` 명령어 등)을 통해 NAS DB 서버로 수만 건 데이터를 1초 만에 꽂아 넣음
    *   *개발 포인트: 기존의 [uploader_cli.py](file:///c:/Users/user/Documents/GitHub/Extrusion_data/uploader_cli.py)를 완전히 대체하는 새롭고 가벼운 콘솔 프로그램*

---

## ⚙️ [CARD 3] 백엔드 (Backend / API Server)
**"창고의 데이터를 예쁘게 가공해서 프론트엔드에게 전달하는 요리사"**

*   **위치:** 클라우드(Vercel, AWS) 또는 로컬 서버
*   **기술 스택:** Node.js (Next.js API Routes, Express), Python (FastAPI) 등 선택
*   **핵심 역할 (R&R):**
    *   **보안과 연결 통제:** 프론트엔드가 NAS DB에 직접 접속(`localhost:5432`)하지 못하도록 중간에서 방패막이 역할 (DB 비밀번호 보호)
    *   **데이터 가공 및 집계:** "최근 1주일간 불량률 통계", "특정 날짜의 온도 그래프 데이터" 등 프론트엔드가 그리기 좋게 데이터를 가공해서 JSON 형태로 전달
    *   **비즈니스 로직 처리:** 로그인 인증, 권한 관리, 사용자별 데이터 접근 제어
    *   *개발 포인트: 기존 Supabase가 공짜로 해주던 REST API(`http://.../rest/v1/...`) 역할을 직접 가벼운 서버로 대체*

---

## 🖥️ [CARD 4] 프론트엔드 (Frontend / Web UI)
**"최종 사용자(조장/관리자)가 조종성과 데이터를 눈으로 보는 화려한 모니터 화면"**

*   **위치:** 웹 브라우저 (PC 정면, 태블릿, 스마트폰 등)
*   **기술 스택:** React, View.js, Svelte, Tailwind CSS 등
*   **핵심 역할 (R&R):**
    *   **시각화 (차트/그래프):** 백엔드에서 받은 가공된 JSON 데이터를 아름다운 대시보드로 렌더링
    *   **사용자 인터랙션:** 작업자가 현장에서 스마트패드로 "작업 완료", "불량 발생" 버튼을 누르는 UI 제공
    *   *절대 하지 않는 역할: DB에 직접 연결하는 코드 작성 금지 (보안 위험)*

### 💡 새로운 프로그램 제작 개발 순서 (추천)
1.  **CARD 1 (DB 띄우기)**: NAS에 터미널 1줄 쳐서 창고부터 준비
2.  **CARD 2 (콜렉터 제작)**: 파이썬 코드로 CSV를 DB에 수만 건씩 꽂아버리는 트럭 코드 짜기 (매우 쉬움)
3.  **CARD 3+4 (백/프론트 제작)**: 그 데이터를 꺼내서 웹에 띄워보는 나만의 대시보드 만들기
